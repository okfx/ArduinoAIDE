qwen3:14b
I can see the issue. In oscilloscope.h, the drawWaveform() function is being called every loop iteration without any frame rate limiting. At high BPMs, the audio ISR fires more frequently, and the SPI transfer to the OLED can't keep up.

The fix is to gate the OLED update behind your existing isSafeToPushOled() check. Here's the change:
oscilloscope.h
void updateOscilloscope(unsigned long nowMs) {
- drawWaveform();
- display.display();
+ if (isSafeToPushOled(nowMs)) {
+ drawWaveform();
+ display.display();
+ }
}
This ensures the oscilloscope only redraws when the OLED bus is free, preventing the flicker at high tempos.
